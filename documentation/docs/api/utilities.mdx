# API: Utilities

SoulState provides utility functions to enhance state management and optimize React component re-renders.

## `shallow`

The `shallow` utility performs a shallow comparison of two objects, checking if their top-level properties are equal. It's primarily used with `useStore` or `store.subscribe()` when selectors return new objects.

### Summary

`shallow` compares the top-level properties of two objects using strict equality (`Object.is`). It returns `true` if all properties are equal, `false` otherwise.

### Function Signature

<CodeBlock language="typescript">
{`function shallow<T, U>(a: T, b: U): boolean;
`}
</CodeBlock>

### Parameters

- `a: T`: The first value to compare
- `b: U`: The second value to compare

### Return Value

- `boolean`: `true` if values are shallowly equal, `false` otherwise

### Usage Examples

#### With `useStore`

Use `shallow` as the third argument to `useStore` when selecting multiple properties.

<CodeBlock language="tsx">
{`import { useStore } from 'soulstate/react';
import { shallow } from 'soulstate/utils';
import { userStore } from '../stores/userStore';

function UserProfile() {
  const { name, email } = useStore(
    userStore,
    (state) => ({ name: state.name, email: state.email }),
    shallow // Only re-render when name or email changes
  );

  return (
    <div>
      <p>Name: {name}</p>
      <p>Email: {email}</p>
    </div>
  );
}
`}
</CodeBlock>

#### With `store.subscribe()`

Use `shallow` with the low-level subscription API.

<CodeBlock language="typescript">
{`import { userStore } from '../stores/userStore';
import { shallow } from 'soulstate/utils';

const unsubscribe = userStore.subscribe(
  (state) => ({ name: state.name, email: state.email }),
  (newData, prevData) => {
    console.log('User data changed:', newData);
  },
  { equalityFn: shallow }
);

// Clean up
unsubscribe();
`}
</CodeBlock>

## `enableInstrumentation`

Expose systems-grade diagnostics to identify propagation bottlenecks and monitor runtime performance.

### Function Signature

<CodeBlock language="typescript">
{`function enableInstrumentation(options?: RuntimeInstrumentation): void;

interface RuntimeInstrumentation {
  onFlush?: (duration: number, keys: Set<DependencyKey>) => void;
  onSelectorRun?: (name: DependencyKey, duration: number) => void;
  onInvalidate?: (key: DependencyKey) => void;
}
`}
</CodeBlock>

### Usage

<CodeBlock language="typescript">
{`store.enableInstrumentation({
  onFlush: (duration, keys) => {
    console.log(\`Flush took \${duration.toFixed(3)}ms for keys:\`, keys);
  },
  onSelectorRun: (name, duration) => {
    console.log(\`Selector \${String(name)} took \${duration.toFixed(3)}ms\`);
  }
});
`}
</CodeBlock>

## `getMetrics`

Retrieves a snapshot of the current runtime performance metrics.

### Function Signature

<CodeBlock language="typescript">
{`function getMetrics(): MetricsSnapshot;

interface MetricsSnapshot {
  flushCount: number;
  selectorRunCount: number;
  invalidationCount: number;
  averageFlushDuration: number;
  totalFlushDuration: number;
}
`}
</CodeBlock>

### Usage

<CodeBlock language="typescript">
{`const metrics = store.getMetrics();
console.log('Average Flush Time:', metrics.averageFlushDuration);
`}
</CodeBlock>

## `objectIs`

The default equality function used by SoulState, providing strict reference equality checking.

### Function Signature

<CodeBlock language="typescript">
{`function objectIs(a: any, b: any): boolean;
`}
</CodeBlock>

### Usage

<CodeBlock language="typescript">
{`import { objectIs } from 'soulstate/utils';

// Same as Object.is
objectIs(1, 1); // true
objectIs(NaN, NaN); // true
objectIs(0, -0); // false

// Reference equality for objects
const obj = {};
objectIs(obj, obj); // true
objectIs({}, {}); // false
`}
</CodeBlock>

## `batch`

Utility for grouping multiple operations. Note that SoulState automatically batches all `setState` calls using microtasks, so this utility is primarily for semantic grouping or advanced patterns.

### Function Signature

<CodeBlock language="typescript">
{`function batch(callback: () => void): void;
`}
</CodeBlock>

### Usage

<CodeBlock language="typescript">
{`import { batch } from 'soulstate/utils';

// Manual batching (rarely needed due to automatic microtask batching)
batch(() => {
  store.setState({ count: 1 });
  store.setState({ user: 'John' });
  store.setState({ active: true });
});
`}
</CodeBlock>

## Performance Characteristics

- **`shallow`**: O(n) where n is number of properties.
- **`objectIs`**: O(1) constant time operation.
- **Automatic batching**: All updates are batched with `queueMicrotask` by the core runtime.

## Best Practices

### ✅ Do:
- Use `shallow` for object selections from multiple properties.
- Use default `Object.is` for primitive values.
- Trust SoulState's automatic microtask batching.
- Import utilities from `'soulstate/utils'`.

### ❌ Don't:
- Use `shallow` for primitive values.
- Expect `shallow` to compare nested objects.
- Manually batch updates unless necessary for code organization.

<Admonition type="info" title="Import Path">
  <p>All utilities are exported from <code>'soulstate/utils'</code> and are also available in the main <code>'soulstate'</code> package.</p>
</Admonition>

<Admonition type="warning" title="Automatic Batching">
  <p>SoulState automatically batches all state updates using microtasks. The <code>batch</code> utility is provided for consistency but is often redundant in practice.</p>
</Admonition>
